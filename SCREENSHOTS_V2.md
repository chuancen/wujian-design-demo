# 新版截图交付说明

本版针对上一版“同系列页面过于雷同”的问题重做。新版不再使用同一模板批量替换标题，而是按 PRD 的 32 个页面逐页设计不同的信息架构和业务界面。

## 输出目录

- `screenshots-v2/方案A-省级政务医疗平台/`
- `screenshots-v2/方案B-实时指挥中心产品/`
- `screenshots-v2/方案C-儿童健康服务品牌产品/`
- `screenshots-v2/方案D-临床采集效率工具/`

## 文件数量

- 每套 32 张 PNG
- 四套共 128 张 PNG
- 每张命名格式：`P编号-页面名称.png`

## 本版改进

- P01 登录/角色入口：多端门户入口，而不是普通后台页。
- P03 组织与角色管理：组织树 + 权限矩阵。
- P06 数据导入结果：导入统计 + 错误行 + 修正建议。
- P08 儿童档案详情：儿童资料 + 历次趋势 + 五健指标。
- P09 档案合并弹窗：重复字段对比和合并策略。
- P11 任务创建向导：步骤条、项目配置、科室人员。
- P13 二维码/条码打印：打印设置 + 一人一码预览。
- P14 医护采集工作台：待检队列、站点进度、设备状态。
- P15 单人采集录入：字段级录入、异常值、范围校验。
- P16 离线同步中心：暂存、待上传、冲突记录。
- P17 设备接入状态：多设备连接状态。
- P18 异常复核：异常名单、规则命中、复核意见。
- P20 随访工作台：按联系状态分列的工作流。
- P21 五健总览：营养、视力、脊柱、口腔、心理专题卡。
- P23-P25 家长端：信息确认、报告、健康教育分别采用服务化页面。
- P26-P28 集成页面：接口配置、数据交换、转诊协同分别独立。
- P29-P31 统计展示：区域驾驶舱、现场调度、只读汇报分别独立。
- P32 系统部署与备份：部署环境、备份恢复、恢复演练。

## 重新生成

```bash
python3 tools/generate_screenshots_v2.py
```

源 SVG 输出到 `screenshot-sources-svg-v2/`，PNG 输出到 `screenshots-v2/`。
